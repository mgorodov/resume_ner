# Тема: Приложения NER для анализа резюме
## Базовый минимум:
Реализовать распознавание именованных сущностей в резюме, сделать замеры метрик качества, обернуть в сервис и навесить интерфейсы.

## Возможные фичи:
- Фильтрации по вакансии/навыкам/локации/..., может даже какое-то ранжирование
- LLM помощник для улучшения резюме
  - как в контексте "напиши шаблон резюме для начинающего DS специалиста" и например под капотом смотреть резюме заточенные под DS и запромптить LLM так чтобы она сформировала шаблон с учетом релевантных скиллов ...
  - так и например: "исправь мое резюме" и получать какой-то новый вариант
  - ...
- Парсить данные с агрегаторов вакансий, например hh и рекомендовать актуальные вакансии по загруженному резюме
- ...

# План работы на год по чекпоинтам:
1. Знакомство с куратором, постановка и уточнение задачи, план на год
2. Сбор данных и EDA\
Сбор данных и их анализ (различные текстовые статистики - распределение сущностей резюме)
3. Бейзлайн
- Получить предсказания максимально простой нейронки без дообучения (библиотека SpaCy)
- Можно добавить ручные правила на регулярках для некоторых сущностей
4. Улучшение бейзлайна\
Дообучить предыдущую модельку на наших данных, можно добавить более сложные модели в тч различные небольшие opensource LLM (детальнее обсудим позднее)
5. Сервис\
Тг бот, Gradio, Streamlit - что-то одно для представления работы
6. Эксперименты с DL
- Дообучение BERT модели (например внутри SpaCy)
- Промпт инжиниринг
- Можно дообучить небольшую LLM (LoRa)
7. Доработка задачи
- Улучшение сервисной части по обратной связи от команды курса по промышленной разработке
- Реализация улучшений на основе фидбека от команды промышленной разработки
